Here we introduce the accompanying R package, gprofiler2 , developed to facilitate programmatic access to g:Profiler computations and databases via REST API. The gprofiler2 package provides an easy-to-use functionality that enables researchers to incorporate functional enrichment analysis into automated analysis pipelines written in R. The package also implements interactive visualisation methods to help to interpret the enrichment results and to illustrate them for publications. In addition, gprofiler2 gives access to the versatile gene/protein identifier conversion functionality in g:Profiler enabling to map between hundreds of different identifier types or orthologous species. The gprofiler2 package is freely available at the CRAN repository .">

Here we introduce the accompanying R package, <b>gprofiler2</b>, developed to facilitate programmatic access to g:Profiler computations and databases via REST API. The <b>gprofiler2</b> package provides an easy-to-use functionality that enables researchers to incorporate functional enrichment analysis into automated analysis pipelines written in R. The package also implements interactive visualisation methods to help to interpret the enrichment results and to illustrate them for publications. In addition, <b>gprofiler2</b> gives access to the versatile gene/protein identifier conversion functionality in g:Profiler enabling to map between hundreds of different identifier types or orthologous species. Usually this involves associating these gene lists with previous knowledge from well curated data sources of biological processes and pathways. However, as the knowledge-bases are constantly changing, keeping the associations up to date requires careful data management. Handling numerous databases, especially when using different gene identifier types, can be a very time-consuming process for researchers.</p><p class="" id=d17051e213>g:Profiler (<a target=xrefwindow href="https://biit.cs.ut.ee/gprofiler" id=d17051e215>https://biit.cs.ut.ee/gprofiler</a>) is a popular web toolset that helps to handle gene lists from various biological and biomedical studies of more than 600 species and strains, including vertebrates, plants, fungi, insects and parasites<sup><a href="#ref-1">1</a>,<a href="#ref-2">2</a></sup>. g:Profilers best known functionality is the over-representation analysis to identify significantly enriched biological functions and pathways obtained from well established data sources which include, among others, Gene Ontology (GO)<sup><a href="#ref-3">3</a></sup>, KEGG<sup><a href="#ref-4">4</a></sup> and Reactome<sup><a href="#ref-5">5</a></sup>. The information about genes, identifier types and GO term associations in g:Profiler is mostly based on Ensembl databases<sup><a href="#ref-6">6</a></sup> including data from Ensembl Genomes, fungi, plants and metazoa specific versions of Ensembl. g:Profiler follows Ensembls quarterly update cycle while keeping the access to previous data versions as archives for reproducibility. The parasite specific data is included from the WormBase<sup><a href="#ref-7">7</a></sup>.</p><p class="" id=d17051e247>Providing users with fast and easy access has been the main goal of g:Profiler developers. Since 2007, g:Profiler has been in constant development and with the recent update in 2019 a new accompanying R package, <b>gprofiler2</b>, was developed<sup><a href="#ref-8">8</a></sup>. The R package relies on the g:Profiler REST API requests providing an easy programmatic access to the same functionalities as in the web tool without performing heavy computations and mappings in R. While there are other popular R packages for functional enrichment analysis, such as <tt>topGO</tt><sup><a href="#ref-9">9</a></sup> and <tt>clusterProfiler</tt><sup><a href="#ref-10">10</a></sup>, <b>gprofiler2</b>, unlike the others, provides access to numerous annotation data sources with a single query without requiring to download any of these sources to a local computer. Furthermore, the mapping between different gene identifiers is automatic and the input can be a mixed list of identifiers. g:Profilers continuous development and flexibility of usage has been recognised by the European Life Science Infrastructure ELIXIR, which has selected it as one of its <a target=xrefwindow href="https://elixir-europe.org/platforms/interoperability/rirs" id=d17051e272>Recommended Interoperability Resources</a>.</p><p class="" id=d17051e276>g:Profiler development team encourages and supports external tools and packages to use either <b>gprofiler2</b> package or the public API to be part of their workflows. For example, <tt>RCAS</tt> Bioconductor package<sup><a href="#ref-11">11</a></sup> includes <b>gprofiler2</b> for functional analysis of transcriptomic regions detected by different high-throughput experiments. Single-cell mapper package (<tt>scMappR</tt>)<sup><a href="#ref-12">12</a></sup> analyses cell-type specific gene lists with <b>gprofiler2</b>. <tt>OmnipathR</tt><sup><a href="#ref-13">13</a></sup> suggests using <b>gprofiler2</b> for enrichment analysis of protein complexes. Gene Co-expression Network analysis pipeline (<a target=xrefwindow href="https://github.com/Kumquatum/GWENA" id=d17051e311>GWENA</a>) uses <b>gprofiler2</b> in their pipeline for functional enrichment of co-expressed gene modules. A Nextflow differential gene expression analysis <a target=xrefwindow href="https://github.com/qbic-pipelines/rnadeseq" id=d17051e318>pipeline</a> includes <b>gprofiler2</b> for pathway analysis.</p><p class="" id=d17051e326>Here we demonstrate how to conveniently incorporate the <b>gprofiler2</b> R package into bioinformatics analysis pipelines using differential gene expression analysis as an example.</p></div><div id=article1-body class=generated-article-body><h2 class=main-title id=d17051e335>Methods</h2><div class=section><a name=d17051e338 class=n-a></a><h3 class=section-title>Implementation</h3><p class="" id=d17051e343>Inherently, <b>gprofiler2</b><sup><a href="#ref-8">8</a></sup> is a collection of wrapper functions in R that simplify sending POST requests to the g:Profiler REST API using the <tt>RCurl</tt> package<sup><a href="#ref-14">14</a></sup>. This means that all the annotation data sources and computations are centralised in a single well-maintained server and therefore the results from both the web tool and R package are guaranteed to be identical. Relying on the central API also simplifies the maintenance of the g:Profiler interfaces and enables the R users to get access to the most up-to-date data without having to download the heavy annotation data files to their own devices.</p><p class="" id=d17051e359>There are four main API wrapper functions in <b>gprofiler2</b>:</p><div class=list><a name=d17051e365 class=n-a></a><ul><li><p id=d17051e369><tt>gost</tt> for functional enrichment analysis</p></li><li><p id=d17051e377><tt>gconvert</tt> for mapping gene identifiers between different namespaces</p></li><li><p id=d17051e385><tt>gorth</tt> for mapping orthologous genes across species</p></li><li><p id=d17051e393><tt>gsnpense</tt> for mapping SNP rs-IDs to chromosome positions, genes and variant effects.</p></li></ul></div><p class="" id=d17051e400>In addition to fetching the results from the API, <b>gprofiler2</b> uses the packages <a target=xrefwindow href="https://cran.r-project.org/package=ggplot2" id=d17051e405>ggplot2</a><sup><a href="#ref-15">15</a></sup> and <a target=xrefwindow href="https://cran.r-project.org/package=plotly" id=d17051e411>plotly</a><sup><a href="#ref-16">16</a></sup> to provide visualisations for enrichment results that are similar to the web tool ones. Using <tt>ggplot2</tt> allows users to customise the visualisations by adding or removing graphical layers, and to adjust the quality of images for publication.</p><p class="" id=d17051e422>This article was written using R version 3.6.1 (2019-07-05) and <b>gprofiler2</b> version 0.1.9.</p></div><div class=section><a name=d17051e429 class=n-a></a><h3 class=section-title>Operation</h3><p class="" id=d17051e434>The <b>gprofiler2</b> R package is <a target=xrefwindow href="https://cran.r-project.org/package=gprofiler2" id=d17051e439>available from CRAN</a> and works on R versions 3.5 and above. The package also includes a detailed <a target=xrefwindow href="https://cran.r-project.org/web/packages/gprofiler2/vignettes/gprofiler2.html" id=d17051e442>vignette</a>.</p><p class="" id=d17051e446>The package can be installed from CRAN:</p><p class="" id=d17051e449><pre class=preformat><span style="font-size:15px;color:#8F5903"># install from CRAN</span>
<span style="font-size:15px;color:#214A87">install.packages</span>(<span style="color:#4F9905">"gprofiler2"</span>)
<span style="font-size:15px;color:#8F5903"># load the package</span>
<span style="font-size:15px;color:#214A87">library</span><span style="font-size:15px;color:#000000">(gprofiler2)</span></pre></p></div><div class=section><a name=d17051e469 class=n-a></a><h3 class=section-title>Input description</h3><p class="" id=d17051e474>The most popular functionality of g:Profiler is functional enrichment analysis provided by the g:GOSt tool that performs over-representation analysis using hypergeometric test. This functionality is available in <b>gprofiler2</b> under the function <tt>gost</tt>. The required inputs for this function are a vector of gene identifiers, <tt>query</tt>, and the name of the corresponding <tt>organism</tt> which is constructed by concatenating the first letter of the genus name and the specific epithet, e.g. <i>hsapiens</i> for human genes. The full list of supported species and strains, 641 in total, is available on the <a target=xrefwindow href="https://biit.cs.ut.ee/gprofiler/page/organism-list" id=d17051e492>g:Profiler web page</a>.</p><p class="" id=d17051e496>The query vector can include mixed types of gene/protein identifiers, SNP rs-IDs, chromosomal intervals or term IDs. Accepting a mixture of IDs is a unique feature that skips time-consuming manual steps of converting between different identifier types required by other functional enrichment tools. However, in case of analysing numeric identifiers (e.g. Entrez IDs) the user should specify the namespace using the <tt>numeric_ns</tt> parameter. The same description of input <tt>query</tt> and <tt>organism</tt> holds for the three other functions in <b>gprofiler2</b>.</p><p class="" id=d17051e511><pre class=preformat><span style="font-size:15px;color:#000000">gostres =</span> <span style="font-size:15px;color:#214A87">gost</span>(<span style="color:#204A87">query = c</span>(<span style="color:#4F9905">"X:1000:1000000"</span>, <span style="color:#4F9905">"rs17396340"</span>, <span style="color:#4F9905">"GO:0005005"</span>,
		           <span style="color:#4F9905">"ENSG00000156103"</span>, <span style="color:#4F9905">"NLRP1"</span>, <span style="color:#4F9905">"3837"</span>),
               <span style="font-size:15px;color:#214A87">organism</span> = <span style="color:#4F9905">"hsapiens"</span>,
               <span style="font-size:15px;color:#214A87">numeric_ns</span> = <span style="color:#4F9905">"ENTREZGENE_ACC"</span>)
</pre></p><p class="" id=d17051e555>Several additional parameters in the <tt>gost</tt> function help to perform the analysis according to specific needs, including custom statistical options such as background definition, statistical significance threshold, method for multiple testing correction and testing for under-representation. Also, additional information like GO evidence codes and genes belonging to the intersection between the input list and the functional term is available.</p></div><div class=section><a name=d17051e562 class=n-a></a><h3 class=section-title>Annotation databases</h3><p class="" id=d17051e567>g:Profilers in-house database includes only reliable annotation data sources that are regularly updated such as Gene Ontology (GO)<sup><a href="#ref-3">3</a></sup>, KEGG<sup><a href="#ref-4">4</a></sup>, Reactome<sup><a href="#ref-5">5</a></sup>, WikiPathways<sup><a href="#ref-17">17</a></sup>, miRTarBase<sup><a href="#ref-18">18</a></sup>, TRANSFAC<sup><a href="#ref-19">19</a></sup>, Human Protein Atlas<sup><a href="#ref-20">20</a></sup>, protein complexes from CORUM<sup><a href="#ref-21">21</a></sup> and Human Phenotype Ontology<sup><a href="#ref-22">22</a></sup>. By default, all the data sources in g:Profiler database are used for the analysis in <b>gprofiler2</b>, but a specific selection can be made with the <tt>sources</tt> parameter of the <tt>gost</tt> function. In order to enable more flexibility, users can also use their own annotation data. The custom source can be uploaded in a Gene Matrix Transposed (GMT) file format. This feature is further described in the next section.</p></div></div><div id=article1-body class=generated-article-body><h2 class=main-title id=d17051e621>Use case</h2><p class="" id=d17051e624>Differential gene expression analysis determines lists of genes that show changes in expression between different conditions, cell types, time points, etc. Functional enrichment analysis using the <b>gprofiler2</b> package<sup><a href="#ref-8">8</a></sup> helps to interpret these gene lists.</p><p class="" id=d17051e634>Here we demonstrate the main functionality of <b>gprofiler2</b> by following an analysis example from the existing RNA-seq Bioconductor workflow<sup><a href="#ref-23">23</a></sup> that uses the popular <tt>DESeq2</tt> package<sup><a href="#ref-24">24</a></sup> for differential analysis. With the recent update, g:Profiler introduced an alternative way for visualising functional terms, a Manhattan plot. On this plot, the x-axis shows the terms and y-axis shows the enrichment P-values on  log<sub>10</sub> scale. Each circle on this plot corresponds to a single term. The circles are colored according to the annotation source and size-scaled according to the total number of genes annotated to the corresponding term. The locations on the x-axis are always fixed and ordered in a way that the terms from the same GO subtree are located closer to each other. This helps to highlight different enriched GO sub-branches as they form peaks in the Manhattan plot and makes plots from different queries easily comparable. For the same reason, by default the values on the y-axis are capped to a maximum value of 16 that corresponds to P-value less than 10<sup>16</sup> . The same default threshold is also used in the statistical tests in R. This selection can be switched off to show the P-values in a wider scale range.</p><p class="" id=d17051e1016>Interactive graphs are common in web tools and therefore the Manhattan plot in g:Profiler web interface also provides several interactive features to facilitate data exploration and enables to export the visualisations as high-quality image files. Mimicking the g:Profiler web interface, the Manhattan plot in <b>gprofiler2</b> is implemented in the function <tt>gostplot</tt> that uses the resulting object from the <tt>gost</tt> function as an input. As a unique feature, compared to other similar packages, the parameter <tt>interactive</tt> enables to switch between interactive <tt>plotly</tt> graph for browsing or static <tt>ggplot</tt> graph for saving as an image file. The parameter <tt>capped</tt> enables to turn off the upper limit of y-axis.</p><p class="" id=d17051e1041><pre class=preformat><span style="font-size:15px;color:#000000">p =</span> <span style="font-size:15px;color:#214A87">gostplot</span><span style="font-size:15px;color:#000000">(gp_up,</span> <span style="font-size:15px;color:#214A87">interactive =</span> <span style="font-size:15px;color:#8F5903">TRUE</span><span style="font-size:15px;color:#000000">)</span></pre></p><p class="" id=d17051e1059>After exploring the interactive graph and deciding on the story to tell about the results, the user can compose a publishable figure that highlights the most important terms using the function <tt>publish_gostplot</tt> and defining the relevant terms in the parameter <tt>highlight_terms</tt>. The users can cite the exact extract of the annotation database and the state of the implementation by stating the version number in their research. class=generated-article-body><h2 class=main-title id=d17051e2955>Conclusion</h2><p class="" id=d17051e2958>We presented the <b>gprofiler2</b> R package<sup><a href="#ref-8">8</a></sup> that is one of the programmatic access points to the widely used <a target=xrefwindow href="https://biit.cs.ut.ee/gprofiler" id=d17051e2967>g:Profiler web toolset</a> for gene list functional enrichment analysis and identifier conversion. This package enables effective integration of g:Profiler functionalities in various bioinformatics pipelines and tools written in R without the need of searching and downloading several data files. The suite of functions in <b>gprofiler2</b> are implemented with the importance of analysis reproducibility and interoperability with other tools in mind. In addition, the package provides a way to easily create or customise the enrichment plots using the existing visualisation packages in R. For the researchers who prefer to perform their computational analysis pipelines through the web we have wrapped the <b>gprofiler2</b> package as a tool for the Galaxy platform<sup><a href="#ref-31">31</a></sup>.</p><p class="" id=d17051e2982>It is important to note that using <b>gprofiler2</b> for functional enrichment analysis is not limited to the use case of differential gene expression analysis. The package is useful whenever there is a set of genes/proteins/SNPs the user wants to characterise with biological functions or to convert to another namespace.</p></div><div id=article1-body class=generated-article-body><h2 class=main-title id=d17051e2991>Data availability</h2><p class="" id=d17051e2994>All data underlying the results are available as part of the article and no additional source data are required.</p></div><div id=article1-body class=generated-article-body><h2 class=main-title id=d17051e3001>Software availability</h2><p class="" id=d17051e3004><b>R package gprofiler2 is available from CRAN:</b> <a target=xrefwindow href="https://cran.r-project.org/package=gprofiler2" id=d17051e3008>https://cran.r-project.org/package=gprofiler2</a>.</p><p class="" id=d17051e3012><b>Source code available from:</b> <a target=xrefwindow href="https://gl.cs.ut.ee/biit/r-gprofiler2" id=d17051e3016>https://gl.cs.ut.ee/biit/r-gprofiler2</a>.</p><p class="" id=d17051e3020><b>Archived source code at time of publication:</b> <a target=xrefwindow href="https://doi.org/10.5281/zenodo.3919795" id=d17051e3024>https://doi.org/10.5281/zenodo.3919795</a><sup><a href="#ref-8">8</a></sup>.</p><p class="" id=d17051e3031><b>License:</b> <a target=xrefwindow href="https://gl.cs.ut.ee/biit/r-gprofiler2/blob/master/LICENSE" id=d17051e3035>GNU General Public License v2.0</a>.</p></div><div id=article1-back class=generated-article-footer><div class=back-section><a name=d17051e3043 class=n-a></a><h2 class=main-title id=d17051e3045>Author information</h2><p class="" id=d17051e3048>LK, UR and IK implemented the package.
